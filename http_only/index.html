Demostración de HttpOnly

Comparación visual entre una cookie expuesta y una cookie protegida contra acceso desde JavaScript

¿Qué es HttpOnly?

HttpOnly es un atributo de seguridad para cookies que evita que JavaScript pueda acceder a ellas mediante document.cookie. Esto ayuda a reducir el riesgo de robo de sesión en ataques XSS.

Escenario vulnerable

Sin HttpOnly JavaScript sí accede

En este caso, la cookie no tiene la protección HttpOnly, por lo que el navegador permite que JavaScript la lea.

Abrir demostración vulnerable

Escenario protegido

HttpOnly activo JavaScript no accede

En este caso, la cookie se envía con HttpOnly, por lo que JavaScript ya no puede leer su valor directamente.

Abrir demostración protegida

Instrucciones

  • En la página vulnerable, document.cookie sí mostrará la cookie.
  • En la página protegida, la cookie HttpOnly no aparecerá al leer document.cookie.
  • En Application → Cookies, revisa la columna HttpOnly.
  • En Network → Headers, revisa el header Set-Cookie.
