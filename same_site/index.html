Demostración de SameSite

¿Qué demuestra este módulo?

SameSite controla si una cookie puede enviarse cuando una petición viene desde otro sitio distinto. En esta versión, la demostración usa:

  • Sitio principal: victima.test:3000
  • Sitio atacante: atacante.test:4000

Así se puede observar correctamente la diferencia entre SameSite=None y SameSite=Strict usando dos sitios distintos.

Escenario vulnerable

SameSite=None Permite cross-site

Aquí la cookie será más permisiva. Después de abrir esta demo, se abrirá un sitio atacante para comprobar si la cookie viaja.

Abrir demo vulnerable

Escenario protegido

SameSite=Strict Bloquea cross-site

Aquí la cookie será más restrictiva. Después de abrir esta demo, se abrirá un sitio atacante para comprobar si la cookie ya no viaja.

Abrir demo protegida

Cómo hacer la prueba correctamente

1. Abre primero el caso vulnerable o protegido.

2. Da clic en “Probar desde sitio atacante”.

3. En la nueva pestaña, pulsa “Lanzar prueba cross-site”.

4. Observa qué cookies recibió el servidor objetivo.

  • Con SameSite=None, la cookie debería viajar.
  • Con SameSite=Strict, la cookie no debería viajar.
